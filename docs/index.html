A repo becomes a queryable graph, then a budgeted answer.

Six ordered stages turn source into signal. No neural embeddings, no tokenizer, no external API. The same corpus produces a byte-identical index every time.

01 / Ingest

Scan

Git-first, gitignore-respected. Two-gate change detection: mtime, then hash.

02 / Parse

Extract

TS/JS compiler AST, tree-sitter for 19 grammars, bespoke extractors for the rest.

03 / Store

Graph

SQLite nodes and edges with provenance, confidence, and FTS5 search.

04 / Resolve

Link

Cross-file import resolution. Anything unproven downgrades to unresolved, honestly.

05 / Rank

Weigh

Relevance x centrality x trust x role. A bounded graph walk from the seed.

06 / Serve

Budget

Position-ordered output across CLI, MCP, and library. Highest signal at the edges.

Read, check, write, verify, apply.

Rinnegan is machinery for AI coding agents, not a graph tool for people. It hands an attention-limited model the exact signal it needs, and nothing else.

understand-> lookup-> write the patch-> verify-> apply

verify returns findings -> back to write. clean -> apply.

What verify actually does: it parses the post-edit image, checks it against the graph for unknown symbols, signature drift, and blast radius, then rolls the overlay back every time. Unknown-symbol and parse errors block the commit. Signature echoes and blast-radius warnings inform, but never fail it.

Twenty-three languages, one code path per grammar.

Composite single-file components (Vue, Svelte, Astro) get their script block sliced out, parsed with the precise TS/JS extractor, and line-remapped back. Non-code is covered too: Markdown links and wikilinks, dependency manifests, and MCP configs. Adding a tree-sitter grammar is one table entry.

Quickstart

Point it at your repo.

# install and build (Node 20+)
npm install
npm run build

# build the knowledge graph
rinnegan index

# get a minimal, provenance-tagged slice
rinnegan understand "how does auth work"

# gate a commit on the graph
rinnegan verify --staged

Any agent, one transport

Run rinnegan install claude (or cursor, codex, kiro, pi, windsurf, gemini) to emit MCP config. rinnegan mcp starts the server over stdio, with understand, lookup, verify, and map.

Share the index in CI

The whole index is one file: .rinnegan/graph.db. Cache it with actions/cache so warm runs update instead of reindexing cold. A status fingerprint confirms two machines hold the same corpus.

Ship it as a pre-commit gate

Wire rinnegan verify --staged into a git hook. Unknown symbols and parse failures block the commit before an edit is ever trusted.

Give your agent the eye.

Deterministic, fully local, and honest about every fact it hands over. Same corpus in, byte-identical index out.
